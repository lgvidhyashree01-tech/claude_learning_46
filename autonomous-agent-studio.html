How the stop-check governs the loop

Architecture overview

Every round is a real, separate API call — no hardcoded sequence and no pre-set round count.

Agent responsibilities

  • Planner — scopes the topic into research questions and a brief structure, once, up front.
  • Executor — runs live web search and writes the first draft. Runs once.
  • Evaluator — scores the current draft against the clarity/structure rubric every round.
  • Critic — turns the Evaluator's score into specific, prioritized fixes every round.
  • Improver — rewrites the full draft using the Evaluator + Critic output. Feeds back into Evaluator — this is the loop.
  • Memory Manager — keeps a condensed, running memory of established facts/sources so the Improver never contradicts or refabricates.
  • Safety Monitor — checks each draft for unreliable sources, unsupported claims, or biased framing, in parallel with scoring.
  • Final Reviewer — runs once, after the loop exits, for a last polish pass and executive summary.

Stop-check, in order, every round

  • 1. Plateau — score improved less than 0.3 for two consecutive rounds. Signals diminishing returns.
  • 2. Threshold — score reaches 9.0/10. The intended, successful ending.
  • 3. Hard cap — 6 rounds reached regardless of score. Safety fallback only, not a goal.

Whichever fires first ends the loop and routes the current draft to the Final Reviewer branch instead of back to the Evaluator.

Ideas & further prompts

Extension ideas

  • Swap the rubric (accuracy, depth, tone) and watch the same loop optimize for something else entirely.
  • Let the user vote thumbs up/down per round and fold that into the Evaluator's next prompt.
  • Add a second Executor pass mid-loop for topics where facts go stale between rounds.
  • Persist the memory notes across sessions so repeat runs on related topics start warm.
  • Add a human-in-the-loop "approve to stop" gate before the Final Reviewer runs.

Prompts to build on this

  • "Add a branch where, if Safety Monitor flags twice in a row, the loop halts for human review instead of continuing."
  • "Make the plateau delta and target score adjustable in the UI before a run starts."
  • "Add a diff view between each round's draft so I can see exactly what the Improver changed."
  • "Turn this into a two-topic comparison mode that runs both pipelines side by side."
